CSS Organisation

The CSS organisation is loosely based on Inverted Triangle CSS by Harry Roberts. The inverted triangle is from the broadest and most applicable styles (variables, base) through to specific modules (objects and components) and then on to very specific trumping utilities and shameful hacks.

  1. Variables: global variables and site-wide settings eg responsive.css
  2. Base: unclassed HTML elements eg a {}
  3. Objects: cosmetic-free objects, abstractions, and design patterns eg o-media.css
  4. Components: discrete, complete chunks of UI eg carousel.css
  5. Utilities: high-specificity, very explicit selectors that trump everything else. Overrides and helper classes eg .u-hiddenvisually {}
  6. Shameful: temporarily disorganised CSS lives in shame.css

The whole idea is that all files relating to a particular module go in that module folder: eg JavaScript, PNGs, SVGs, CSS and the Handlebars template.
